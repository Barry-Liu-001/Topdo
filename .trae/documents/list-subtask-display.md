# 列表增加子项展示

## 需求分析

当前任务列表中，`TaskItem` 卡片仅在 meta 行显示子任务进度（如 `2/5`），用户需要展开详情面板才能查看和操作子任务。本次需求是在列表视图中直接展示子任务项，让用户无需展开详情即可查看和勾选子任务。

## 现状

- **数据模型**：`Task` 接口已有 `sub_tasks?: SubTask[]` 字段，`SubTask` 包含 `id`、`text`、`done`、`created_at`
- **卡片视图**：仅在 `task-meta-line` 中显示 `subTaskDone/subTaskTotal` 进度文本
- **详情面板**：展开后可查看/编辑/添加/删除子任务

## 实现方案

在 `TaskItem.vue` 的任务卡片（`.task-card`）下方，增加子任务列表区域，支持折叠/展开和直接勾选。

### 步骤 1：修改 TaskItem.vue 模板 — 添加子任务展示区域

在 `.task-card` 的 `</article>` 之后、`<div v-if="inlineEditing">` 之前，添加子任务列表区域：

- 仅在 `subTaskTotal > 0` 时显示
- 每个子任务显示：勾选框 + 文本（已完成的加删除线）
- 点击勾选框可直接切换子任务完成状态（调用 `toggleSubTask` + 自动保存）
- 子任务区域带有缩进，视觉上与父任务卡片区分

### 步骤 2：修改 TaskItem.vue 逻辑 — 支持列表中直接切换子任务

- 新增 `toggleSubTaskInline(id: string)` 方法：切换子任务状态后自动调用 `saveDetailDrafts()` 保存
- 复用现有的 `subTasks`、`subTaskDone`、`subTaskTotal` 计算属性
- 复用现有的 `toggleSubTask` 方法逻辑，但增加自动保存行为

### 步骤 3：修改 TaskItem.vue 样式 — 子任务列表样式

- 子任务区域在卡片下方，左侧缩进对齐
- 每个子任务行：小型圆形勾选框 + 文本
- 已完成子任务：文本灰色 + 删除线
- 勾选框 hover 效果
- 子任务区域与卡片之间有视觉连接（左侧竖线或缩进线）

### 步骤 4：调整卡片 meta 行

- 保留 `subTaskDone/subTaskTotal` 进度文本，但改为可点击，点击后展开/收起子任务列表
- 或者直接移除 meta 行中的进度文本，因为子任务已直接展示（保留进度文本更直观）

## 涉及文件

| 文件 | 改动内容 |
|------|---------|
| `src/components/TaskItem.vue` | 模板：添加子任务列表区域；逻辑：添加 `toggleSubTaskInline` 方法；样式：子任务列表样式 |

## UI 设计要点

```
┌─────────────────────────────────────┐
│ ○  任务名称                    10:30 │  ← 任务卡片
│    📅 明天  2/5                     │  ← meta 行（保留进度）
├─────────────────────────────────────┤
│  ○ 子任务1                          │  ← 子任务列表（新增）
│  ● 子任务2（已完成，删除线）         │
│  ○ 子任务3                          │
└─────────────────────────────────────┘
```

- 子任务区域与卡片视觉上属于同一组（共享圆角底部）
- 左侧有缩进线连接，表示层级关系
- 勾选框尺寸小于主任务的勾选框（14px vs 18px）
